# Git best practices

Status: Initial outline

Guidance for keeping repository history understandable and reviewable.

## Expected content

- Commit message conventions.
- Branch naming.
- Pull request readiness.
- Review-friendly change sizing.
- Handling generated files.

## Starter checklist

- [ ] Keep commits focused.
- [ ] Review diffs before staging.
- [ ] Avoid mixing formatting-only changes with behavior changes.
- [ ] Keep secrets and local configuration out of commits.
